# Научный клубок

Задайте вопрос — получите связный ответ по корпусу научно-технической
документации со ссылками на конкретные эксперименты и страницы источников.

Решение для трека «Научный клубок» — Норникель AI Science Hack 2026.

---

## Что это даёт

- **Вопрос → ответ с цитатами.** Спросите «какие методы обессоливания воды
  подходят при сульфатах 200–300 мг/л» — получите текст ответа + ссылки на
  документы + подграф связанных сущностей.
- **Литобзор и сравнение.** Один клик — получите обзор мировой практики с
  разделением РФ / зарубеж, или сравните два материала по всем свойствам.
- **Карта пробелов.** Тепловая карта «материал × процесс» показывает, что
  уже изучено, а какие клетки пустые — куда двигать исследования.
- **Экспорт.** Ответ забирается в Markdown, JSON-LD (FAIR) или PDF.
- **Локально или в облаке.** LLM — Ollama на вашей GPU, либо Yandex
  Foundation Models. Anthropic/OpenAI не нужны.

## Быстрый старт

Нужен **Docker Desktop** и (для локального режима) **Ollama** с GPU.

```powershell
git clone <репозиторий>
cd scientific-tangle
copy .env.example .env

# Модели ~12 ГБ. Только если запускаете LLM локально.
ollama serve
ollama pull qwen2.5:14b-instruct-q5_K_M
ollama pull qwen2.5:3b-instruct-q5_K_M

docker compose up -d
```

Первый запуск — 5–15 минут (сборка образов, автоматический прогон корпуса
из `data/corpus/` при пустой БД).

Откройте:

- **Приложение:** <http://localhost:3000>
- API Swagger: <http://localhost:8010/docs>
- Neo4j Browser: <http://localhost:7474> (`neo4j` / `changeme123`)

## Что попробовать первым делом

1. Откройте <http://localhost:3000>.
2. Нажмите на любой из четырёх **примеров вопросов** — сразу пойдёт ответ.
3. Пока LLM пишет ответ, справа появится **подграф** — кликните на узел,
   чтобы увидеть карточку сущности.
4. Внизу под ответом — список **источников** с номером страницы и типом
   документа. Нажмите «Открыть файл документа» — откроется исходный PDF/DOCX.
5. Кнопка **PDF** справа сверху — экспорт ответа в брендированный PDF.

Дальше — навигация в шапке:

| Раздел | Что там |
|--------|---------|
| **Вопрос** | Главная. Задать вопрос, выбрать формат (обычный / литобзор / сравнение) и гео-фильтр (РФ, зарубеж, всё). |
| **Пробелы** | Тепловая карта «материал × процесс». Белые ячейки — неизученные комбинации. Клик по ячейке — LLM формулирует гипотезу «почему пусто». |
| **Сравнение** | Выберите два материала или два режима — получите таблицу свойств с разбросом значений. |
| **Эксплорер** | Прогулка по графу. Начните с материала → соседние эксперименты → авторы → журналы. |
| **Статистика** | Сколько чего в базе, распределения по годам, географии, типам документов. |
| **Данные** | Прогресс обогащения графа локальной LLM и метрики качества данных. |

## Свой корпус документов

Положите файлы в `data/corpus/` и рестартните backend
(`docker compose restart backend`). Поддерживаются:

- PDF, DOCX, DOC, PPTX, PPT, XLSX, XLS, TXT
- вложенные архивы `.zip`, `.rar`, многотомные `.001-.002` — распаковываются
  рекурсивно, русские имена в cp866 читаются корректно
- бинарные DOC/PPT/XLS конвертируются через LibreOffice, если он есть в
  контейнере (уже добавлен в Dockerfile)

Рекомендуемая структура — из неё автоматически извлекаются тип документа,
издание и год:

```
data/corpus/
└── Источники информации/
    ├── Доклады/…                    → doc_type = report
    ├── Журналы/<Издание>/<Год>/…    → doc_type = journal (+ journal, year)
    ├── Статьи/…                     → doc_type = article
    ├── Обзоры/…                     → doc_type = review
    └── Материалы конференций/…      → doc_type = conference
```

При пустой БД backend сам прогонит `data/corpus/` при старте
(`AUTO_INGEST_ON_STARTUP=true`) и запустит фоновое обогащение через LLM.
Прогресс — на странице **Данные**.

## Локальная LLM или Yandex

Переключается одной переменной `LLM_PROVIDER` в `.env`:

| Провайдер | Плюсы | Минусы |
|-----------|-------|--------|
| `ollama` | Всё локально, данные не покидают машину, бесплатно | Нужна NVIDIA GPU и ~15 ГБ VRAM |
| `yandex` | Не нужна GPU, быстрее на слабом железе | Нужен API-ключ Yandex Foundation Models, есть rate-limit |

Эмбеддинги переключаются независимо через `EMBEDDING_PROVIDER`
(`yandex` / `local` / `ollama`). Полный список настроек — в `.env.example`.

## Если что-то не работает

**Первый ответ идёт 30–60 секунд.** Нормально: Qwen прогревается в VRAM,
fastembed качает модель эмбеддингов. Дальше — быстро.

**Ответ пустой или «ничего не нашлось».** Откройте страницу **Данные** —
если extract и enrich ещё не закончили работу, граф пока разреженный.
Обогащение 10 000 записей занимает несколько часов (14B) или ~1 час (3B).

**HTTP 500 в ответе.** Обычно значит что backend перезагружается или
обогащение забило LLM. Подождите 10 секунд и перезадайте вопрос.

**Ollama недоступен.** На хосте запустите `ollama serve` — backend ходит
на `host.docker.internal:11434`. Проверить: `curl http://localhost:11434/api/tags`.

**Хочу качество лучше.** Поставьте `PREMIUM_MODE=true` в `.env` — включится
Qwen 32B (нужно ≥ 24 ГБ VRAM). Или переключитесь на `LLM_PROVIDER=yandex`.

## Стек

- **Backend:** Python 3.11 · FastAPI · pdfplumber · python-docx · reportlab
- **Frontend:** Next.js 15 · Cytoscape.js · Chart.js · Tailwind
- **Граф:** Neo4j 5 + APOC + Graph Data Science
- **Векторный поиск:** Qdrant
- **LLM:** Ollama (Qwen 2.5 14B + 3B) или Yandex Foundation Models
- **Эмбеддинги:** `multilingual-e5` через fastembed, либо Yandex, либо bge-m3 через Ollama
- **Auth:** JWT HS256 + PBKDF2, audit-log в SQLite
